# Lesson Loom Agent Eval Checklist

Passes if:

- Names the learner and promise.
- Chooses a useful first artifact.
- Produces or critiques a 5-8 lesson spine.
- Explains the pain chain between lessons.
- Identifies a cut list and next-course ladder.
- Includes an eval path, not just course prose.
- Separates public site content from durable framework artifacts.
- Escalates to Course Builder only when concrete drafting is needed.
- `agent.md` is portable Pi-agent contract; `AGENTS.md` is only the OpenClaw adapter.

Fails if:

- It treats course examples as the canonical framework home.
- It leaves evals as an afterthought.
- It lets personalization/model comparison bloat the first course.
- It accepts a toy demo with no real nail.
- It makes OpenClaw-specific paths or tools required for the portable agent.
