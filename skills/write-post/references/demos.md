# Demos

Use demonstrations only when they clarify the article's core idea.

## Prefer
- Mermaid architecture graphs for systems posts
- small code blocks for setup or usage
- tiny before/after examples
- one sharp visual that carries a point

## Avoid
- decorative images with no job
- too many diagrams
- diagrams that repeat the prose

## Rule

If the demo does not teach faster than prose, leave it out.
