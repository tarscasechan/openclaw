# Want / Need / Get

Use this as an internal structural frame before drafting. It is scaffolding, not post-body format.

## Want
- Open with the bold statement.
- Create curiosity.
- Name the reader's desire or frustration.

## Need
- Explain what has to change.
- Show why the old shape fails.
- Name the pain points plainly.

## Get
- Show the new shape.
- Say what it is.
- Say what it is not.
- Mention new tradeoffs or problems it introduces.

## Pass check
- Want names the reader's desire or frustration.
- Need names the failure in the current shape.
- Get names the mechanism, not just the promise.
- Tradeoffs are stated plainly instead of hidden. If there is no tradeoff, the brief is not done.

## Use it early
- Apply this before sentence-level editing.
- If the structure is weak, restart from concept.
- Do not publish `Want`, `Need`, or `Get` as literal section headings unless the user explicitly asks for a process note or brief.
- In drafts, translate the frame into a natural lead, argument, and payoff.
