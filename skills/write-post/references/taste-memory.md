# Taste Memory

Use this as medium-confidence taste guidance for `write-post`. It records confirmed user preferences from a focused review pass, not universal writing law.

Primary sources:

- `logs/write-post-taste/user-review-2026-04-30.json`
- `plans/write-post-taste-followup.md`

## Confidence

- Treat these rules as active guidance for posts in the current voice family.
- Do not overfit one answer into a permanent prohibition.
- Older posts written before the current skill chain are weaker taste evidence than recent rewrites.

## Demos

Keep demos that teach judgment mechanics, system relationships, or executable prompts faster than prose.

- Keep useful prompt blocks when the reader can reuse them.
- Keep compact decision trees when they clarify a real judgment rule.
- Simplify diagrams that have the right idea but too much process machinery.
- Do not cut a demo merely because it exposes process; cut or simplify it when the process bulk overwhelms the point.

Examples:

- Keep the counterexample prompt block because it changes what the reader can do next.
- Keep the ask-versus-act decision tree because it explains a judgment rule faster than prose.
- Simplify the writing-workflow gate diagram because the backtracking idea matters, but the diagram is heavy.

## Hooks

Prefer hooks with a turn over descriptive openings that only set atmosphere.

- A `but` turn can make a personal discovery hook sharper.
- Aphoristic hooks need exact articles and nouns; small wording changes matter.
- Lyrical scene-setting should not delay the argument unless the scene creates direct tension.

Examples:

- Strong direction: `I tried to make the agent smarter, but the answer was already sitting in Contacts.app.`
- Strong direction: `A happy path is not an automation.`
- Weak pattern: vivid facts that feel like atmosphere before the real argument arrives.

## Metaphors

Protect compact metaphors only when they reveal a real system relationship.

- Good lines are not sacred just because they sound polished.
- For personal AI plumbing, the wall metaphor may carry more than plumbing alone: plumbing belongs inside the walls; paint, wallpaper, and decor belong on the outside.
- Prefer metaphors that show where complexity should live.

## Agent And Operator Language

Write about agent/human collaboration without overexplaining the human role.

- Keep the balance between agent and human operator implicit when possible.
- For direct line edits in this family, prefer phrases like `shared workspace between agent and operator` when they preserve the partnership without stopping to define it.
- Do not erase the agent/operator relationship into vague phrases like `where the work happens`.
- Avoid making the post announce that it is balancing agent and human unless the post is directly about that balance.

## Editing Check

During Zinsser and read-aloud passes, ask:

- Did this cut remove a useful turn?
- Did this demo get simpler, or merely vanish?
- Is this compact line carrying a relationship, or just sounding clever?
- Is the agent/operator balance present without being overexplained?
