# Research Pain Points

## Goal

Collect the real frustrations behind the topic.

## Output

Return:
- the failure mode
- who feels it
- what it costs
- what people already tried
- why that did not solve it
- author stake: what the writer saw, felt, tried, broke, learned, or now believes

Use those labels plainly when extracting notes so the result can move cleanly into packaging.

## Rules

- Prefer concrete failures over broad themes.
- Pull in negative experience and objection.
- Preserve lived friction. If the user gives personal experience, keep it as `author stake` instead of generalizing it into "users".
- If no author stake is available, label that as missing rather than inventing it.
- If the pain is fuzzy, ask for more detail.
- Do not package or draft yet.
