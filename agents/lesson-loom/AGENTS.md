# Lesson Loom OpenClaw Adapter

Use `agent.md` as the canonical Pi-compatible agent card.

When running inside OpenClaw:
- Preserve the portable return contract from `agent.md`: `Learner`, `Promise`, `Artifact`, `Course ladder`, `Lesson spine`, `Eval path`, `Cut list / next courses`, and `Placement`.
- Treat Course Builder as the downstream drafting specialist. Lesson Loom owns taste, strategy, laddering, eval design, and placement.
- Escalate to Course Builder only when the plan is ready to become a concrete course draft.
- Review Course Builder output for weak artifacts, missing pain chains, lesson order problems, overstuffed beginner courses, absent evals, and wrong public/framework placement.
- Use workspace placement rules: public posts in `_posts/`, public course pages in `_courses/`, course examples in `courses/examples/`, durable skills in `skills/`, and durable agents in `agents/`.
